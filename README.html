Introduction

Text files consisting mostly of word lists (whitespace-delimited/line-based entries) GNU Aspell, CSV (Comma Separated Values) Wikipedia, Office Open XML OOXML (a.k.a. ECMA-376 ECMA) and similar data sets that are useful in manually crafting test cases or developing an automated black-box fuzzer. In particular, the data contained within these mostly text files is intended for the purposes of programmatically testing security..

Inspirations

If you're already familiar with established repositories such as fuzzdb and SecLists, then werdlists is quite similar in mission as it's a centralized attack strings resource--just with its own unique style and organization, original hand-crafted contents, expanded concepts, etc.

Hierarchy

The scripts folder consists of shell scripts used for maintenance and such. Data folders are outlined in the root-index.lst file at the root of the repository's directory hierarchy. All files in each folder are detailed in one of their own specific index.lst files which is like root-index.lst, but it only contains entries that describe associated file names as opposed to folder names.

Naming

Although the majority of files have the .txt extension that signifies the MIME type text/plain--pay close attention, because other file extensions are in use on occassion as well, such as .csv (comma-separated values) Wikipedia, or .xml which denotes Extended Markup Language. Files with a non-ASCII or binary character set are assigned the .bin extension.
